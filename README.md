# wdio-ts-cucumber-skeleton
A scalable UI automation skeleton built with WebdriverIO, TypeScript, Cucumber (BDD), and Allure reporting. Designed for robust, reusable, enterprise-grade test automation.
This repository provides a clean and extensible UI automation skeleton using WebdriverIO, TypeScript, and Cucumber (BDD). It follows a modular Page Object Model (POM) with a shared BasePage, offering a strong foundation for building enterprise-ready automation frameworks.

# The project includes:

WebdriverIO (latest version)

TypeScript with strict typing

Cucumber BDD with feature-wise structure

Page Object Model with BasePage utilities

Centralized configuration setup

Allure reporting integration

Clean folder structure suitable for any UI automation project

Reusable commands and OOP-style design

Ready for CI/CD integration (GitHub Actions, Jenkins, Azure DevOps, etc.)

Ideal for organizations looking to standardize UI automation across teams and scale reliably.
